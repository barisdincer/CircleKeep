# Security Policy

CircleKeep handles contact and call-log metadata, so privacy and local data safety are treated as security concerns.

Please do not report sensitive issues in a public GitHub issue. Use GitHub Security Advisories if available, or contact the maintainer privately.

Useful reports include:

- Permission bypasses or confusing permission flows.
- Contact or call-log data leaving the device unexpectedly.
- Backup import crashes or unsafe parsing behavior.
- Database migration bugs that can delete or corrupt local data.
- Any future network or sync behavior that weakens the local-first privacy model.

CircleKeep is currently pre-1.0. Security fixes may be released without a fixed schedule while the project is still taking shape.
